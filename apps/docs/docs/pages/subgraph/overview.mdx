# Subgraph

The 0xSlots subgraph indexes all slot deployments, ownership transitions, tax events, and metadata updates. It's the primary data source for reading historical and aggregate slot data.

[Source](https://github.com/nezz0746/0xSlots/blob/main/packages/subgraph/schema.graphql)

## Entities

### `Factory`

The factory contract state.

| Field | Type | Description |
|---|---|---|
| `id` | `ID` | Factory proxy address |
| `slotCount` | `BigInt` | Total deployed slots |
| `modules` | `[Module]` | Registered modules |

### `Account`

An address that has interacted with slots.

| Field | Type | Description |
|---|---|---|
| `id` | `ID` | Address (lowercase) |
| `type` | `AccountType` | `EOA`, `CONTRACT`, `DELEGATED`, or `SPLIT` |
| `slotCount` | `Int` | Slots created as recipient |
| `occupiedCount` | `Int` | Currently occupied slots |
| `slotsAsRecipient` | `[Slot]` | Derived — slots where this is the recipient |
| `slotsAsOccupant` | `[Slot]` | Derived — slots where this is the occupant |

### `Currency`

An ERC-20 token used by slots.

| Field | Type | Description |
|---|---|---|
| `id` | `ID` | Token address |
| `name` | `String` | Token name |
| `symbol` | `String` | Token symbol |
| `decimals` | `Int` | Token decimals |

### `Slot`

The core entity. Represents one slot contract.

| Field | Type | Description |
|---|---|---|
| `id` | `ID` | Slot contract address |
| `recipient` | `Bytes` | Revenue recipient |
| `currency` | `Currency` | Payment token |
| `mutableTax` | `Boolean` | Whether tax can change |
| `mutableModule` | `Boolean` | Whether module can change |
| `manager` | `Bytes` | Manager address |
| `taxPercentage` | `BigInt` | Current tax rate (bps/month) |
| `module` | `Module` | Active module (if any) |
| `occupant` | `Bytes` | Current occupant (null = vacant) |
| `price` | `BigInt` | Self-assessed price |
| `deposit` | `BigInt` | Current deposit |
| `collectedTax` | `BigInt` | Uncollected tax |
| `totalCollected` | `BigInt` | Lifetime collected tax |
| `createdAt` | `BigInt` | Creation timestamp |
| `metadata` | `MetadataSlot` | Derived — metadata from MetadataModule |

Plus derived relations to all event types: `purchases`, `releases`, `liquidations`, `settlements`, `taxCollections`, `priceUpdates`, `deposits`, `withdrawals`, `metadataUpdates`.

### `MetadataSlot`

Metadata attached to a slot via the MetadataModule.

| Field | Type | Description |
|---|---|---|
| `id` | `ID` | Slot address |
| `uri` | `String` | Current metadata URI |
| `rawJson` | `String` | Full JSON content (from IPFS or inline) |
| `adType` | `String` | Parsed `type` field from JSON (e.g. "banner", "native") |
| `updatedBy` | `Bytes` | Last updater address |
| `updateCount` | `BigInt` | Total URI changes |

### `Module`

A registered module contract.

| Field | Type | Description |
|---|---|---|
| `id` | `ID` | Module address |
| `verified` | `Boolean` | Whether the module is verified |
| `name` | `String` | Module name |
| `version` | `String` | Module version |

## Event Entities

All events are **immutable** (append-only). Each includes `slot`, `timestamp`, `blockNumber`, and `tx`.

| Entity | Key Fields |
|---|---|
| `SlotDeployedEvent` | `recipient`, `currency`, `manager`, `deployer`, tax/module config |
| `BoughtEvent` | `buyer`, `previousOccupant`, `price`, `deposit`, `selfAssessedPrice` |
| `ReleasedEvent` | `occupant`, `refund` |
| `LiquidatedEvent` | `liquidator`, `occupant`, `bounty` |
| `PriceUpdatedEvent` | `oldPrice`, `newPrice` |
| `DepositedEvent` | `depositor`, `amount` |
| `WithdrawnEvent` | `occupant`, `amount` |
| `SettledEvent` | `taxOwed`, `taxPaid`, `depositRemaining` |
| `TaxCollectedEvent` | `recipient`, `amount` |
| `TaxUpdateProposedEvent` | `newPercentage` |
| `ModuleUpdateProposedEvent` | `newModule` |
| `PendingUpdateCancelledEvent` | — |
| `MetadataUpdatedEvent` | `author`, `uri`, `rawJson`, `adType` |

## NFT Entities

For ERC721Slots (Harberger NFT collections):

| Entity | Description |
|---|---|
| `NFTCollection` | A collection with name, symbol, currency, tax rate |
| `NFTToken` | Individual token linked to a Slot |
